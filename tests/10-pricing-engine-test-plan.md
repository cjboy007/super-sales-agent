# 定价引擎测试方案 (pricing-engine)

---

## 测试目标

验证智能定价策略的准确性、规则引擎正确性和利润计算精确性。

---

## 模块概述

- 产品成本数据
- 客户分级数据
- 市场竞争数据（可选）

## 2. 集成测试

### 2.1 完整定价流程

**测试场景：** 基于客户和产品自动定价

**测试数据：**
```json
{
  "customer": {
    "customer_id": "okki_12345",
    "tier": "A",
    "history": {
      "total_orders": 50,
      "total_amount": 500000
    }
  },
  "products": [
    {"sku": "PROD-001", "cost": 5.00, "quantity": 500}
  ],
  "rules": {
    "margin_target": 0.30,
    "volume_discount": true,
    "customer_discount": true
  }
}
```

**预期流程：**
1. 获取产品成本
2. 应用目标毛利
3. 应用数量折扣
4. 应用客户折扣
5. 计算最终价格
6. 验证利润

**通过标准：** 价格准确，利润达标

### 2.2 与报价单集成

**测试场景：** 定价引擎输出到报价单

**预期流程：**
1. 定价引擎计算价格
2. 输出到 quotation-workflow
3. 生成报价单
4. 价格一致

**通过标准：** 价格传递正确

### 2.3 与 OKKI 集成

**测试场景：** 客户历史数据获取

**预期流程：**
1. 查询 OKKI 客户数据
2. 获取订单历史
3. 计算客户等级
4. 应用对应折扣

**通过标准：** 客户数据准确

## 3. 端到端测试

### 3.1 E2E-001: 标准报价定价

**步骤：**
1. 客户询价
2. 获取产品信息
3. 查询客户等级
4. 计算价格
5. 生成报价单
6. 发送客户

**预期结果：** 价格有竞争力且利润达标

### 3.2 E2E-002: 大客户定价

**步骤：**
1. 大客户询价
2. 识别 VIP 身份
3. 应用最大折扣
4. 验证最低利润
5. 生成报价

**预期结果：** 大客户获得优惠价格

### 3.3 E2E-003: 竞争定价

**步骤：**
1. 获取竞争对手价格
2. 调整定价策略
3. 保持利润底线
4. 生成有竞争力报价

**预期结果：** 价格有竞争力

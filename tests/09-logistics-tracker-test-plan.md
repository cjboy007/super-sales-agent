## 2. 集成测试

### 2.1 完整物流追踪流程

**测试场景：** 发货后自动追踪

**测试数据：**
```json
{
  "shipment": {
    "tracking_no": "1234567890",
    "carrier": "DHL",
    "order_no": "ORD-20260327-001",
    "ship_date": "2026-03-27",
    "destination": "New York, USA"
  }
}
```

**预期流程：**
1. 订单发货
2. 录入物流单号
3. 自动查询状态
4. 每 6 小时更新
5. 签收后通知

**通过标准：** 状态实时更新

### 2.2 与订单追踪集成

**测试场景：** 物流状态同步到订单

**预期流程：**
1. 物流状态更新
2. 自动同步到 order-tracker
3. 订单状态更新
4. 通知客户

**通过标准：** 状态同步及时

### 2.3 异常预警集成

**测试场景：** 物流异常自动告警

**预期流程：**
1. 物流显示异常
2. 自动创建告警
3. 通知客服
4. 联系物流公司

**通过标准：** 异常及时处理

## 3. 端到端测试

### 3.1 E2E-001: 标准物流追踪

**步骤：**
1. 订单发货
2. 录入 DHL 单号
3. 系统自动追踪
4. 客户查看状态
5. 签收确认

**预期结果：** 全程可追踪

### 3.2 E2E-002: 物流异常处理

**步骤：**
1. 物流显示延误
2. 系统自动告警
3. 客服联系客户
4. 解释原因
5. 继续追踪

**预期结果：** 异常妥善处理

### 3.3 E2E-003: 多包裹追踪

**步骤：**
1. 订单分多个包裹
2. 录入多个单号
3. 分别追踪
4. 汇总状态

**预期结果：** 多包裹正确追踪

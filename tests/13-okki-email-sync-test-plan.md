## 2. 集成测试

### 2.1 完整邮件同步流程

**测试场景：** 邮件发送后同步到 OKKI

**测试数据：**
```json
{
  "email": {
    "from": "sales@farreach.com",
    "to": "customer@acme.com",
    "subject": "Quotation QT-20260327-001",
    "timestamp": "2026-03-27T10:00:00Z"
  }
}
```

**预期流程：**
1. 邮件发送
2. 提取收件人域名
3. 匹配 OKKI 客户
4. 创建跟进记录
5. 关联相关文档
6. 记录已处理

**通过标准：** 同步准确及时

### 2.2 双向同步测试

**测试场景：** OKKI → 本地 和 本地 → OKKI

**预期流程：**
1. OKKI 客户更新 → 本地同步
2. 本地邮件发送 → OKKI 同步
3. 数据一致性验证

**通过标准：** 双向一致

### 2.3 与邮件处理集成

**测试场景：** 邮件处理完成后自动同步

**预期流程：**
1. 邮件处理完成
2. 触发同步
3. 写入 OKKI
4. 确认成功

**通过标准：** 自动同步

## 3. 端到端测试

### 3.1 E2E-001: 新邮件同步

**步骤：**
1. 收到新客户邮件
2. 创建 OKKI 客户
3. 写入跟进记录
4. 后续邮件关联

**预期结果：** 新客户完整记录

### 3.2 E2E-002: 历史邮件导入

**步骤：**
1. 导入历史邮件
2. 批量匹配客户
3. 批量写入跟进
4. 去重处理

**预期结果：** 历史数据完整

### 3.3 E2E-003: 多用户同步

**步骤：**
1. 多个销售发送邮件
2. 分别同步到 OKKI
3. 关联到对应销售
4. 权限正确

**预期结果：** 多用户正确隔离

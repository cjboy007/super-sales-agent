# 售后管理测试方案 (after-sales)

---

## 测试目标

验证售后服务流程的完整性、问题追踪准确性和客户满意度管理有效性。

---

## 模块概述

**核心文件：**
- `skills/after-sales/cli/after_sales_cli.js` - CLI 工具
- `skills/after-sales/cli/okki_sync_cli.js` - OKKI 同步
- `skills/after-sales/api/controllers/okki_sync_controller.js` - API 控制器
- `skills/after-sales/test/e2e_test.js` - 端到端测试
- `skills/after-sales/test/okki_sync_test.js` - 同步测试

**依赖：**
- OKKI CRM API
- 订单追踪模块

## 2. 集成测试

### 2.1 完整售后流程

**测试场景：** 客户投诉处理

**测试数据：**
```json
{
  "ticket": {
    "ticket_id": "AFT-20260327-001",
    "order_no": "ORD-20260327-001",
    "customer_id": "okki_12345",
    "type": "quality_issue",
    "priority": "high",
    "description": "产品表面有划痕",
    "evidence": ["photo1.jpg", "photo2.jpg"]
  }
}
```

**预期流程：**
1. 客户投诉
2. 创建工单
3. 分配负责人
4. 调查问题
5. 提供解决方案
6. 客户确认
7. 关闭工单
8. 满意度调查

**通过标准：** 全流程完整

### 2.2 与订单追踪集成

**测试场景：** 订单关联售后

**预期流程：**
1. 订单状态 → delivered
2. 客户发起售后
3. 关联原订单
4. 查看订单详情
5. 处理售后

**通过标准：** 订单关联正确

### 2.3 与物流追踪集成

**测试场景：** 物流问题售后

**预期流程：**
1. 客户反馈未收到
2. 调用 logistics-tracker
3. 查询物流状态
4. 确认问题
5. 补发或退款

**通过标准：** 物流查询正确

## 3. 端到端测试

### 3.1 E2E-001: 质量问题处理

**步骤：**
1. 客户收到货发现质量问题
2. 发送照片投诉
3. 创建售后工单
4. 质量部门调查
5. 提供解决方案（补发/退款）
6. 客户确认
7. 执行方案
8. 满意度调查

**预期结果：** 问题妥善解决

### 3.2 E2E-002: 物流丢失处理

**步骤：**
1. 客户反馈未收到
2. 查询物流状态
3. 确认丢失
4. 联系物流公司索赔
5. 补发给客户
6. 关闭工单

**预期结果：** 物流问题正确处理

### 3.3 E2E-003: 退货流程

**步骤：**
1. 客户申请退货
2. 审核退货原因
3. 批准退货
4. 提供退货地址
5. 收到退货
6. 退款
7. 关闭工单

**预期结果：** 退货流程完整

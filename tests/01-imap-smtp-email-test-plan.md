# 邮件处理模块测试方案 (imap-smtp-email)

---

## 测试目标

验证 IMAP/SMTP 邮件自动收发、附件下载、邮件捕获功能的正确性、稳定性和安全性。

---

## 模块概述

**核心文件：**
- `skills/imap-smtp-email/auto-capture.js` - 邮件自动捕获
- `skills/imap-smtp-email/scripts/imap.js` - IMAP 读取
- `skills/imap-smtp-email/scripts/smtp.js` - SMTP 发送
- `skills/imap-smtp-email/intent-recognition.js` - 意图识别
- `skills/imap-smtp-email/reply-generator.js` - 回复生成
- `skills/imap-smtp-email/okki-sync.js` - OKKI 同步

**依赖：**
- IMAP 服务器 (imaphz.qiye.163.com)
- SMTP 服务器 (smtp.qiye.163.com)
- OKKI CRM API

## 2. 集成测试

**测试场景：** 完整邮件处理流程

**测试数据：**
```json
{
  "incoming_email": {
    "from": "customer@example.com",
    "subject": "询价：产品 A",
    "body": "你好，请问产品 A 的单价是多少？我们需要 5000 个。",
```

**预期流程：**
1. IMAP 捕获新邮件

7. 写入 OKKI 跟进记录

**通过标准：** 全流程无异常，OKKI 有跟进记录

## 3. 端到端测试

### 3.1 E2E-001: 完整询价处理流程

**前置条件：**
- IMAP/SMTP 配置正确
- OKKI API 可用
- 知识库有产品数据

**步骤：**
1. 外部发送询价邮件到系统邮箱
2. 系统自动捕获邮件
3. 识别为 price_inquiry
4. 检索知识库生成报价
5. 自动回复客户
6. 写入 OKKI 跟进记录

**预期结果：**
- 客户收到回复邮件（< 5 分钟）
- OKKI 中有跟进记录
- 本地有邮件归档

**通过标准：** 全流程自动化，无人工干预

### 3.2 E2E-002: 投诉邮件升级处理

**步骤：**
1. 发送投诉邮件（含"投诉"、"质量问题"关键词）
2. 系统识别为 complaint
3. 置信度 > 0.9 → 自动标记高优先级
4. 发送 Discord 通知人工介入
5. 人工审核后发送回复

**预期结果：** 高优先级投诉正确升级

### 3.3 E2E-003: 多轮对话上下文保持

**步骤：**
1. 客户发送第一封邮件（询价）
2. 系统回复报价
3. 客户回复同一线程（还价）
4. 系统识别为同一对话
5. 保持上下文生成回复

**预期结果：** 上下文正确关联，回复连贯

- `skills/follow-up-engine/test/e2e.sh` - 端到端测试

**依赖：**
- OKKI CRM API
- Cron/Heartbeat 系统

## 2. 集成测试

### 2.1 完整跟进流程

**测试场景：** 报价单发送后自动跟进

**测试数据：**
```json
{
  "event": "quotation_sent",
  "customer_id": "okki_12345",
  "quotation_no": "QT-20260327-001",
  "timestamp": "2026-03-27T10:00:00Z"
}
```

**预期流程：**
1. 监听报价单发送事件
2. 创建跟进任务（2 天后）
3. 到期时生成提醒
4. 发送 Discord 通知
5. 销售跟进后更新状态

**通过标准：** 提醒准时发送

### 2.2 多事件联动

**测试场景：** 客户生命周期多阶段跟进

**事件序列：**
1. 新客户创建 → 3 天跟进
2. 发送报价 → 2 天跟进
3. 客户回复 → 1 天跟进
4. 发送样品 → 5 天跟进
5. 样品确认 → 发送订单

**通过标准：** 各阶段跟进正确

## 3. 端到端测试

### 3.1 E2E-001: 新客戶培育流程

**步骤：**
1. OKKI 创建新客户
2. 系统自动创建跟进任务
3. 3 天后发送提醒
4. 销售发送开发信
5. 记录跟进结果

**预期结果：** 新客户正确培育

### 3.2 E2E-002: 报价跟进流程

**步骤：**
1. 发送报价单
2. 2 天后提醒跟进
3. 销售联系客户
4. 客户反馈记录到 OKKI

**预期结果：** 报价跟进及时

### 3.3 E2E-003: 样品跟进流程

**步骤：**
1. 样品发出
2. 5 天后提醒确认
3. 销售询问反馈
4. 客户确认/修改

**预期结果：** 样品反馈及时收集

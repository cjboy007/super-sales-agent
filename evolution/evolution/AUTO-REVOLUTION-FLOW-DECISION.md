# 🤖 Auto Revolution 流程选择指南

**创建日期：** 2026-03-29  
**用途：** 帮助用户选择合适的执行流程（完整流程 vs 简化流程）

---

## 📋 流程类型对比

| 流程类型 | 步骤 | 模型配置 | 耗时 | 成本 | 适用场景 |
|----------|------|----------|------|------|----------|
| **简化流程** | Executor 直连 | bailian/qwen3.5-plus | ~5 分钟/任务 | 🆓 免费 | 任务明确、低风险、批量执行 |
| **完整流程** | Reviewer → Executor → Auditor | Gemini 3.1 Pro → Qwen3.5-Plus → Gemini 3.1 Pro | ~15 分钟/任务 | 🆓 免费 | 复杂任务、高风险、关键功能 |
| **高级流程** | Reviewer → Executor → Auditor | Sonnet-4.6 → GPT-5.4 → Sonnet-4.6 | ~15 分钟/任务 | 💰$$ | 核心功能、安全敏感、生产发布 |

## 🔍 任务分析维度

### 1. 任务复杂度

| 等级 | 描述 | 推荐流程 |
|------|------|----------|
| **简单** | 单一功能、代码量<100 行、文档更新 | 简化流程 |
| **中等** | 多文件修改、代码量 100-500 行、需要测试 | 简化流程 或 完整流程 |
| **复杂** | 跨模块修改、代码量>500 行、架构调整 | 完整流程 或 高级流程 |

### 2. 风险评估

| 风险等级 | 描述 | 推荐流程 |
|----------|------|----------|
| **低风险** | 不影响现有功能、可回滚、仅文档 | 简化流程 |
| **中风险** | 影响部分功能、需测试验证 | 完整流程 |
| **高风险** | 核心功能、安全相关、不可回滚 | 高级流程 |

### 3. 时间敏感性

| 紧急程度 | 描述 | 推荐流程 |
|----------|------|----------|
| **紧急** | 需要立即完成、Bug 修复 | 简化流程 |
| **正常** | 按计划执行、功能改进 | 完整流程 |
| **不紧急** | 长期优化、技术债务 | 高级流程 |

### 4. 质量要求

| 质量等级 | 描述 | 推荐流程 |
|----------|------|----------|
| **标准** | 功能正常、基本测试 | 简化流程 |
| **严格** | 完整测试、文档齐全 | 完整流程 |
| **生产级** | 零缺陷、审计追踪 | 高级流程 |

                    风险等级

                ┌─────────────────┐
              简 │ 简化 │ 简化 │ 完整 │ 紧急
                ├─────────────────┤
              正 │ 简化 │ 完整 │ 完整 │ 正常

              不 │ 简化 │ 完整 │ 高级 │ 不紧急
                └─────────────────┘
                  低    中    高

## 🎯 典型场景推荐

### 场景 1：文档更新
- **任务：** 更新 SKILL.md、添加参考文档
- **复杂度：** 简单
- **风险：** 低
- **推荐：** 简化流程 ✅

### 场景 2：新功能实现
- **任务：** 实现 forward 命令、draft-edit 命令
- **复杂度：** 中等
- **风险：** 中
- **推荐：** 完整流程 ✅

### 场景 3：安全修复
- **任务：** 修复 prompt injection 漏洞、添加安全规则
- **复杂度：** 中等
- **风险：** 高
- **推荐：** 高级流程 ✅

### 场景 4：代码重构
- **任务：** 统一命令命名、状态码对齐
- **复杂度：** 中等
- **风险：** 中（需向后兼容）
- **推荐：** 完整流程 ✅

### 场景 5：核心架构
- **任务：** 修改邮件发送核心逻辑、会话头维护
- **复杂度：** 复杂
- **风险：** 高
- **推荐：** 高级流程 ✅
